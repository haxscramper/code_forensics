Detailed Description

template<std::integral IdType>
struct dod::Id< IdType >

DOD Id type

Note
It does not have a default constructor, if you need co construct a an empyt/nil value (not recommended) for some field/argument you can use the static ::Nil method defined in the derivations produced in the DECL_ID_TYPE macro
ID types can be thought of as a pointers or indices into some 'memory' container and as such should only be parametrized using unsigned data types. They do not provide any sort of pointer-like arithmentics or any other operations that might fail due to the overflow. Container indices can be freely converted to the pointers.
